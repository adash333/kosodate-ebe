# Webアプリ用 新規リポジトリ作成手順

子育てエビデンスチャットボットを、**この Obsidian vault とは別の Git リポジトリ**で
管理するための手順書。データ源（vault）とアプリ（新repo）を分離します。

GA4 測定ID: **`G-N6B7HX8CQ8`**（控え済み）

---

## 0. 全体の考え方（なぜ分けるか）

| リポジトリ | 役割 | 公開設定 |
|---|---|---|
| `adash333/ObsidianGitVault`（既存） | 論文・動画の**元データ（md）**。非公開の作業場 | Private |
| 新リポジトリ（これから作る） | **公開Webアプリ**のコード＋生成済み `advice.json` | Public |

- データは「一度生成して固定」なので、vault から `advice.json` を**書き出して
  新repoにコピー**する方式が一番シンプル（submodule等は使わない）。
- vault を Public にせずに済む（元データ・下書きを公開しなくてよい）。

## 5. データの受け渡し（vault → 新repo）

`advice.json` は vault の md から生成します。生成バッチ
（`webapp/scripts/build-data.mjs`、計画書M1で作成予定）を **vault側で実行** し、
出力された `advice.json` を新repoの `src/data/` にコピーします。

```bash
# vault側で（例）
node webapp/scripts/build-data.mjs > /tmp/advice.json
# 新repo側へコピー
cp /tmp/advice.json ../kosodate-ebe/src/data/advice.json
```

> 新作動画が増えたら、この生成→コピーを再実行するだけで更新できます。
> （手順は新repoのREADMEにも書いておきます）

## 6. GA4（アクセス解析）の組み込み

`src/analytics.ts` を作り、測定ID `G-N6B7HX8CQ8` で初期化します。

```html
<!-- index.html の <head> に追加（最も簡単な方法） -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-N6B7HX8CQ8"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-N6B7HX8CQ8');
</script>
```

- 「どの悩みカテゴリが選ばれたか」などは `gtag('event', ...)` で送信できます
  （実装時にこちらで仕込みます）。

- `public/CNAME`（中身 `k.5micro.net`）を置く。

- DNSは **CNAMEレコード `k` → `adash333.github.io`** を1件追加。
- 詳細は既存の `webapp/デプロイ手順.md` をそのまま流用（新repoにコピー）。
- `vite.config.ts` の `base` は、独自ドメイン直下配信なので `'/'` でOK。

## 9. このリポジトリ（vault）側でやること

- `webapp/` 配下の**計画・手順・legal草案・build-dataスクリプト**は当面 vault に

  ポインタ（「アプリは kosodate-ebe リポジトリへ移管」）だけ残す方針も可。

## 10. Claude Code に新repoで作業させる方法（重要）

現在のセッションは GitHub 操作が `adash333/ObsidianGitVault` に限定されており、
**新repoの作成・pushはこのセッションからはできません**。新repoを使うには:

1. 上記 手順2 で**あなたが新repoを作成**する（数クリック）。
2. その後、**新repoをスコープに含めた新しいセッション**（Claude Code on the web で
   対象リポジトリに `adash333/kosodate-ebe` を選ぶ）を開始する。
3. 新セッションで「`webapp/` の計画書に沿ってM1から実装して」と指示すれば、
   こちらでコード一式（UI・ロジック・advice.json・GA4・Pages設定）を作成・push します。

> つまり、**あなたの作業は「手順2でrepoを作る」だけ**。中身の実装は次セッションで
> こちらが進めます。advice.json の生成は vault のmdを参照する必要があるため、
> 新セッションに vault も一緒にスコープ追加できると理想的です（両repo選択）。
