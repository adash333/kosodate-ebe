# お問い合わせフォーム送信有効化手順

## 背景

会社情報ページの運営サイト一覧から「眼科医からみた子供の視力回復法」を削除した。

お問い合わせフォームは画面上には表示されているが、送信先URLが未設定の場合は
「お問い合わせフォームは現在準備中です。しばらくお待ちください。」と表示される。

## 現在の実装

- フォームコンポーネント: `src/components/ContactForm.tsx`
- 表示場所:
  - `src/components/Company.tsx`
  - `src/components/Contact.tsx`
- 送信先環境変数: `VITE_CONTACT_ENDPOINT`
- GitHub Actionsでは `secrets.VITE_CONTACT_ENDPOINT` を `npm run build` に渡す設定済み。

## 送信できるようにする手順

1. Google Apps Scriptなどでフォーム受信用のWebアプリURLを作成する。
   - `POST` を受け取る。
   - メール送信またはスプレッドシート保存を行う。
   - デプロイ時は「実行ユーザー: 自分」「アクセス: 全員」にする。

2. GitHubリポジトリにSecretを追加する。
   - Repository: `adash333/kosodate-ebe`
   - `Settings` → `Secrets and variables` → `Actions`
   - `New repository secret`
   - Name: `VITE_CONTACT_ENDPOINT`
   - Secret: Google Apps ScriptのWebアプリURL

3. GitHub Actionsを再実行する。
   - `Actions` → `Deploy to GitHub Pages` → `Run workflow`
   - または次回push時に自動デプロイされる。

## 注意点

`VITE_` で始まる環境変数はブラウザ側に公開されるため、秘密トークンは入れない。
スパム対策は、現在のフォームにあるハニーポット項目 `company` や、GAS側の簡易レート制限で行う。
