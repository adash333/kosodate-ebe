# お問い合わせページ削除とmain直push運用

## 概要

独立した「お問い合わせ」ページ（`/contact`）を削除し、以降の開発は
フィーチャーブランチを使わず直接 `main` ブランチへ push する運用に切り替えた。
あわせてビルド成果物 `tsconfig.tsbuildinfo` を Git の追跡対象から除外した。

## 実施内容

### 1. お問い合わせページの削除

会社情報ページ（`/company`）の確認後、独立した `/contact` ページの削除を依頼された。

- `src/App.tsx`
  - `Contact` コンポーネントの import を削除
  - `/contact` ルート分岐を削除
  - トップ画面フッターおよび `Shell` フッターの「お問い合わせ」リンクを削除
- `src/components/Contact.tsx` を削除
- `src/components/ContactForm.tsx` を削除（`Contact.tsx` からのみ参照されていた）

※ 会社情報ページ（`src/components/Company.tsx`）内の「お問い合わせ」セクションは
外部フォーム（`https://m.5micro.net/about`）への案内のため残している。

型チェック（`tsc -b`）が通ることを確認。

### 2. main 直 push 運用への切り替え

ユーザーより「今後も直接 main にお願いします」と明示的な許可を受領。
フィーチャーブランチ `claude/company-info-page-pbkdkq` の変更を `main` に
fast-forward マージして push。以降の作業は `main` で直接行う。

### 3. tsconfig.tsbuildinfo の追跡除外

`git add -A` により誤って追跡されていたビルド成果物を整理。

- `git rm --cached tsconfig.tsbuildinfo` で追跡解除
- `.gitignore` に `*.tsbuildinfo` を追加

## 関連コミット（main）

- お問い合わせページ削除（Contact / ContactForm / ルート / フッターリンク）
- tsconfig.tsbuildinfo を追跡対象から除外（.gitignore 更新）

## 今後の運用メモ

- 開発・push は `main` ブランチへ直接行う。
- ビルド成果物（`*.tsbuildinfo` 等）はコミットしない。
