# 動画リストを投稿日ベースの公開判定に更新

## 概要

YouTube動画リスト（`data/advice.json`）を更新。従来の `MAX_VIDEO_ID`（番号しきい値）による
公開判定をやめ、ObsidianGitVault `0 YouTube動画リスト.md` の **投稿日** をもとに
「投稿日 < 実行日(JST)」で公開判定する方式に変更した。2026-06-19 時点で **154件 → 159件**。

※ この作業の実体（コード変更・再生成）は機能ブランチ `claude/yesterdays-report-yo6oyr` に
コミット済み（レビュー前のため main 未マージ）。本ログのみ main に直 push している。

## 背景・きっかけ

- ユーザー依頼「動画リストの更新をお願いできますか？まずは提案して下さい」。
- 調査の結果、データは #194 まで仕込み済みだが `MAX_VIDEO_ID = 156` で公開が止まっていた。
- さらにマスターリストを確認すると **id順 ≠ 投稿順**（例: #165 は 5/6 投稿、#122 は 6/18 投稿、
  逆に #123/#125/#126/#150 は id が小さいのに投稿日が 6/25〜6/28 と未来）。
  番号しきい値では「投稿日が本日より前」を正しく表現できないと判明。
- ユーザー指示「投稿日が、本日より前の動画をお願いします」「厳密適用（未来日付の4件も除外）」。

## 実施内容

### 1. 投稿日台帳の追加
- `data/video-publish-dates.json`（新規）… id→YouTube投稿(予定)日（#001〜#270、270件）。
  出典は ObsidianGitVault `0 YouTube動画リスト.md`。投稿日をリポジトリ内で再現可能にした。

### 2. merge-data.mjs を投稿日ベースに変更
- `scripts/merge-data.mjs`
  - `MAX_VIDEO_ID = 156` 方式を廃止。
  - `video-publish-dates.json` を読み、実行日(JST)を算出し
    「投稿日 < 実行日」の動画のみ `advice.json` に含める。
  - 投稿日不明のidは安全側で除外（ログに警告出力）。

### 3. advice.json 再生成（154→159件）
- 追加(9): #157〜#164（6/10〜6/17）、#165（就活ep1, 5/6）
- 除外(4): #123/#125/#126/#150（投稿日が 6/25〜6/28 の未来）
- #166（本日6/19）以降は未投稿として除外。就活ep2-4(#172-174)・近視(#178) は
  従来どおり `enrichment.json` で除外設定済みのため非掲載。

### 4. README 更新
- `README.md` … データ件数・データ更新手順を投稿日ベースに更新、
  `video-publish-dates.json` の説明を追記。

## 今後の運用メモ

- 新作の投稿日が決まったら `data/video-publish-dates.json` に追記し
  `node scripts/merge-data.mjs` を再実行 → `advice.json` 再生成。
  番号しきい値の手動更新は不要になった（投稿日到来で自動的に掲載対象になる）。
- マスターリスト側の既知の不整合（#178 の重複行・タイトルずれ等）は未対応。必要時に整理する。
