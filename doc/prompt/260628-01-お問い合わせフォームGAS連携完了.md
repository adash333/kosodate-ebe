- **使用Googleアカウント**: 5micromember@gmail.com（GAS作成・デプロイ・通知受信）

- フロント `src/components/ContactForm.tsx` は完成済み
  （POST送信・ハニーポット `company`・トークン `VITE_CONTACT_TOKEN` 送信対応済み）
- **deploy.yml に `VITE_CONTACT_TOKEN` の受け渡しが抜けていた**（`VITE_CONTACT_ENDPOINT` のみ）→ 修正

## 反映済みアクション

### リポジトリ変更（main直push）
- `gas/Code.gs` 追加: POST受信 → ハニーポット判定 → トークン検証 → スプレッドシート追記 → メール通知（`doPost`）。動作確認用 `doGet` 付き
- `gas/README-deploy.md` 追加: 5micromember@gmail.com でのデプロイ手順書
- `.github/workflows/deploy.yml` 修正: `VITE_CONTACT_TOKEN` をビルドへ受け渡し
- 空コミットで再デプロイをトリガー（Secret登録後の再ビルド用）

### ユーザー操作（GAS / GitHub / 動作確認）
- Googleスプレッドシート「子育てエビデンス相談室 お問い合わせ」を新規作成し、拡張機能→Apps Scriptに `Code.gs` を貼り付け
- ウェブアプリとしてデプロイ（実行ユーザー: 自分 / アクセス: 全員）
- `doGet` で疎通確認 OK（`{"ok":true,"message":"kosodate-ebe contact endpoint is alive"}`）
- GitHub Secret 2つ登録: `VITE_CONTACT_ENDPOINT`（ウェブアプリURL）、`VITE_CONTACT_TOKEN`（合言葉）
- 本番 https://5micro.net/contact/ からテスト送信 → **3点すべて成功**
  - フォーム送信完了メッセージ表示
  - 通知メール着信（件名「【子育てエビデンス相談室】お問い合わせ」）
  - スプレッドシートに行追記

## 運用メモ

- 合言葉トークンを変えたいときは GAS の `SHARED_TOKEN` と GitHub Secret `VITE_CONTACT_TOKEN` の両方を同じ新値に更新する
- GASコード修正時は「デプロイ → デプロイを管理 → 編集 → バージョン:新バージョン」。**ウェブアプリURLは変わらない**ためSecret再設定は不要
- 問い合わせ者がメールを記入した場合、通知メールの「返信」でそのまま返信できる（`replyTo` 設定済み）
