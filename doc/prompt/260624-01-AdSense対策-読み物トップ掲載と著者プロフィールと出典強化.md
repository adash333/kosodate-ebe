# AdSense再申請対策: 読み物のトップ掲載・お問い合わせ復活・著者プロフィール・出典強化

## 概要

Google AdSense に「有用性の低いコンテンツ」として却下された件への対策として、サイトの
信頼性（YMYL/E-E-A-T）とクローラビリティを高める一連の改修を `main` に直 push（本番反映）した。
Gemini からのレビュー指摘（YMYLの壁／動的Webアプリの構造／記事数・出典）を踏まえ、
プリレンダリング（前段で実施済み）に加えて、トップへの読み物掲載・お問い合わせフォーム復活・
著者プロフィール追加・各出典への原典検索リンク追加を実施した。

## 実施内容

### 1. トップページに「新着の読み物」を掲載（クローラビリティ・低価値対策）

- `src/App.tsx`: intro（トップ）に `新着の読み物（エビデンス解説）` セクションを追加。
  公開済み記事を新しい順に6本、タイトル・リード文・所要時間つきで本文入り表示。
  末尾に「読み物の一覧をすべて見る →」導線も追加。
- `src/styles.css`: `.intro-articles` `.intro-more` のスタイルを追加。
- プリレンダリングにより、トップの静的HTMLに独自記事の本文と内部リンクが含まれ、
  「ボタン1つだけの薄いページ」という誤認を回避する。

### 2. お問い合わせページ／フォームを復活

- `src/components/Contact.tsx`, `src/components/ContactForm.tsx` を git 履歴（削除前）から復元。
- `src/App.tsx`: `/contact` ルートを再追加。
- `src/components/SiteFooter.tsx`: フッターに「お問い合わせ」リンクを再追加。
- `src/entry-server.tsx`: `/contact` をプリレンダリング対象に追加（title/description 付与）。
- `src/components/Company.tsx`: お問い合わせ導線を外部リンク（m.5micro.net）から
  サイト内 `/contact` に変更（自己完結した問い合わせ手段）。
- 送信先は既存どおり `VITE_CONTACT_ENDPOINT`（GitHub Actions secret）。
  ※secret 未設定だと送信時に「準備中」表示。手順は `doc/prompt/260610-01` 参照。

### 3. 著者プロフィール「この記事を書いた人」を追加（YMYL/E-E-A-T 強化）

- `src/author.ts`（新規）: 執筆・監修者情報を集約。
  - 名前: りさお（活動名・本名は非公開）／肩書き: 医師・医学博士
  - ブログ「東大理三夫婦の子育て」(https://risan.jpn.org/) と
    YouTube「ゆっくり子育てエビデンス研究所」(https://www.youtube.com/@evilab) への外部リンクで実在性を担保。
- `src/components/AuthorCard.tsx`（新規）: 記事末尾・運営者情報で共有する著者カード。
- `src/components/Articles.tsx`: 全記事末尾に `AuthorCard` を表示。
  曖昧な「監修: 子育てエビデンス研究所」表記は削除し「運営: 合同会社5マイクロ」に。
- `src/components/Company.tsx`: 「執筆・監修者」セクションを追加し同じプロフィールを掲載。
- `src/styles.css`: `.author-card` 系のスタイルを追加。

### 4. 各出典に原典検索リンク（Google Scholar）を追加（出典の明記 #1）

- `src/components/Articles.tsx`: 「参考にした研究」の各出典に、書誌（著者名・年）から
  生成した Google Scholar 検索リンク「原典を探す」を併記。冒頭に説明注記も追加。
  日本語解説記事（risan.jpn.org）に加えて原典（英語論文）への到達経路を明示。
- データ変更なし・表示ロジックのみで全26記事に自動適用。
- `src/styles.css`: `.refs-note` `.ref-sep` `.ref-scholar` を追加。
- PubMed 直リンクは PMID 誤紐付けリスクがあるため不採用。Scholar 検索方式を採用。

## 今後の運用メモ

- デプロイ後、Search Console で `https://5micro.net/sitemap.xml` を送信し、
  トップ・主要記事を「URL検査 → インデックス登録をリクエスト」。
  インデックスが進んでから AdSense 再申請する。
- お問い合わせフォームの送信可否（`VITE_CONTACT_ENDPOINT` secret）を実機で確認する。
- 重要記事のみ、必要に応じて出典を PubMed/DOI 直リンクへ手動差し替え可能。
