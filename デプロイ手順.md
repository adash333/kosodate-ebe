## 現在の構成（2026-06-10〜）

1. **ビルド＆デプロイ**: `main` へ push すると GitHub Actions（`.github/workflows/deploy.yml`）が
   `npm run build` → GitHub Pages へ自動デプロイ
2. **カスタムドメイン**: `public/CNAME` の中身は `5micro.net`（1行）
3. **DNS**（さくらインターネットのドメインコントロールパネル → 5micro.net → ゾーン情報）:

   | エントリ名 | タイプ | 値 | 用途 |
   |---|---|---|---|
   | `@` | A | `185.199.108.153` | GitHub Pages（apex） |
   | `www` | CNAME | `@` | apexと同じ向き先 |
   | `@` | MX | `1 SMTP.GOOGLE.COM.` | メール（変更しない！） |
   | `@` | TXT | `v=spf1 ...` / `google-site-verification=...` | メール・サイト確認（変更しない） |

   - GitHub推奨のAレコードは4本（`185.199.108.153` / `.109.153` / `.110.153` / `.111.153`）。
     さくらの画面で1本しか入らない場合でも1本で動作する（冗長性が下がるだけ）。
   - 旧 `k` の CNAME（`adash333.github.io.`）は不要。残っていても害はないが削除してよい。
4. **HTTPS**: GitHub → Settings → Pages → Custom domain `5micro.net`、
   「Enforce HTTPS」**ON**（設定済み）

## よくある変更

### アプリを更新する

`main` に push するだけ（数分で反映）。データ更新の手順は `README.md` を参照。

### ドメインを変えたくなったら

1. `public/CNAME` を新ドメインに書き換えて push
2. DNS側で新ドメインを GitHub Pages へ向ける
   （apex → Aレコード4本 ／ サブドメイン → CNAME `adash333.github.io`）
3. Settings → Pages で Custom domain を入れ直し、証明書発行後に Enforce HTTPS を ON

### トラブル時

- **旧サイトが表示される** → DNSキャッシュ。時間を置くか、スマホ回線・シークレットウィンドウで確認
- **Enforce HTTPS が「Unavailable」** → 証明書発行待ち（15分〜1時間）。
  1時間以上続くなら Custom domain を Remove → 再入力で再トリガー
- **404になる** → Actions の deploy が緑か、`public/CNAME` が正しいかを確認

## 注意

- DNSの **MXレコード・TXTレコードは絶対に消さない**（メールが止まる）
- 審査中（AdSense等）はサイトを Unpublish しない
