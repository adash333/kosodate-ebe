# デプロイ手順（GitHub Pages ＋ k.5micro.net）

このアプリを `https://k.5micro.net` で公開するための手順。DNS設定が初めてでも
進められるよう、順番に書いています。**実際の作業はアプリ完成後（M4）に行います。**

## 全体像（4ステップ）

1. GitHub Actions でアプリをビルドし、GitHub Pages へ自動デプロイ
2. リポジトリに `CNAME` ファイル（中身 `k.5micro.net`）を置く
3. 5micro.net のDNSに「CNAMEレコード」を1件追加（← ここがDNS作業）
4. GitHub Pages 側でHTTPS（鍵マーク）を有効化

---

## ステップ3：DNSレコードの追加（いちばん分からない所を詳しく）

### そもそもDNSとは

「`k.5micro.net` という住所に来た人を、GitHubのサーバーへ案内する」ための
転送設定です。これを **CNAMEレコード** という1行で表します。

追加するのは、たった1件、次の内容です。

| 種類(Type) | 名前(Name/ホスト) | 値(Value/向き先) |
|---|---|---|
| `CNAME` | `k` | `adash333.github.io` |

- **名前(Name)** は `k` だけでOKな管理画面が多いです（`k.5micro.net` と
  フルで書く画面もあります。その場合はフルで）。
- **値(Value)** は GitHubユーザー名 `adash333` ＋ `.github.io`。末尾のドット
  `adash333.github.io.` を求める画面もあります（あればドット付きで）。

### 手順（5micro.net を管理しているサービスの画面で）

1. 5micro.net を契約・管理しているサービス（お名前.com / ムームードメイン /
   Cloudflare / Xserver など）にログインします。
   → **どのサービスで5micro.netを取りましたか？教えていただければ、その
     サービス専用の「どのボタンを押すか」まで具体的に案内します。**
2. 「DNS設定」「DNSレコード」「ゾーン編集」などのメニューを開きます。
3. 「レコードを追加」を選び、上の表のとおり入力します
   （Type=CNAME, Name=k, Value=adash333.github.io）。
4. TTLは初期値（自動／3600など）のままでOK。保存します。
5. 反映に数分〜最大1日かかることがあります（通常は数分）。

### 注意

- CNAMEは「サブドメイン（k.〜）」だから使えます。ルート(`5micro.net`)に
  CNAMEは付けられないので、必ず `k` を頭に付けます。
- 既に `k` のレコードがある場合は重複できません（その時は相談してください）。

---

## ステップ4：GitHub側のHTTPS有効化

1. GitHubのリポジトリ → Settings → Pages を開く。
2. 「Custom domain」に `k.5micro.net` と入力して保存（CNAMEファイルが自動生成）。
3. DNSが反映されると「Enforce HTTPS」にチェックできるようになるので、ON。
   → これで `https://k.5micro.net` が鍵マーク付きで表示されます。
