# 子育てエビデンスチャットボット 計画書（v1）

作成日: 2026-06-07 ／ チャンネル「東大理三夫婦の子育てエビデンス研究所」付帯Webアプリ

## 0. このドキュメントの位置づけ

YouTubeチャンネルの約163本（論文ベース動画）の知見を、子育て中の親が
「悩みを選んでいくだけ」でエビデンスに基づく一言アドバイスを受け取れる
Webアプリ（チャットボット風）にする計画書。実装前の合意形成用。

## 1. 確定済みの方針（ユーザー確認済み）

| 項目 | 決定 |
|---|---|
| 頭脳の仕組み | **ルールベース（静的・無料）**。実行時にAI APIを呼ばない |
| ホスティング | **GitHub Pages + 独自サブドメイン `k.5micro.net`**（静的・無料、CNAME設定） |
| 結果に含める | ①一言アドバイス ②関連YouTube動画リンク ③論文の出典 ④次の一歩 |

| 年齢 | **最後のQ4で年齢帯**（未就学／小学生／中高生）を聞いて結果を絞る |

| リポジトリ | アプリは**別の新規リポジトリ**で管理（vault=データ源/Private、新repo=アプリ/Public）。詳細は `webapp/新規リポジトリ作成手順.md` |

## 2. コンセプト / ユーザー体験

「何に関するお悩みですか？」から始まり、**3クリック＋最後に年齢**（各3〜5択）
または**自由入力**で、最後に「今のあなたの子育てが少し幸せになる一言」を
1〜数枚のカードで返す。

### 流れ（3クリック → 最後に年齢）

```
[Q1] だれの・なんのお悩み？          （大カテゴリ 6択）
   例: 学習・勉強 / 行動・しつけ / 心・自己肯定感 / 発達特性 /
       生活習慣・環境 / 家族・きょうだい
        ↓
[Q2] もう少し具体的には？            （Q1に応じた 3〜5択）
   例（学習）: 勉強しない / 宿題 / 集中力 / ほめ方で伸ばしたい / スマホ
        ↓
[Q3] いまの状況に近いのは？          （3〜5択 ＋「うまく言えない→自由入力」）
        ↓
[Q4] お子さんはどれくらい？          （3択・最後に聞く）
   未就学（〜6歳） / 小学生 / 中高生        ← 結果を年齢帯で絞り込み
        ↓
[結果カード] 一言アドバイス＋次の一歩＋論文出典＋YouTubeリンク（1〜3枚）
```

- 年齢は**最後のQ4**で聞き、各エントリの `ageBands` で結果を重み付け（合致を優先表示）。

- 各画面に**自由入力欄**を常設（「選択肢にない／うまく言えない」人向け）。
  自由入力はキーワード一致でタグ検索 → 該当カードを提示（LLM不使用）。
- 結果は「断定」せず、「研究ではこう示されています」というトーンで提示。

## 3. システム構成

- **フロント**: React + Vite（TypeScript）。単一ページSPA、バックエンドなし。
- **データ**: ビルド時に同梱する静的 `advice.json`（後述のデータ整備で生成）。
- **マッチング**: 決定木のリーフ → カテゴリ/タグ集合。エントリをタグ重なりで
  スコアリングして上位提示。自由入力は簡易キーワード一致（タイトル・タグ・知見文）。
- **ホスティング**: GitHub Pages。本リポジトリの GitHub Actions で `webapp/` を
  ビルドし、成果物を Pages へデプロイ。
- **独自サブドメイン**: `k.5micro.net` で**確定**。
  - リポジトリに `CNAME` ファイル（中身 `k.5micro.net`）を置く。
  - 5micro.net のDNSに **CNAMEレコード `k` → `adash333.github.io`** を追加
    （設定手順は `webapp/デプロイ手順.md` を参照。ユーザーはDNS未経験のため要サポート）。
  - GitHub Pages 側で HTTPS（Let's Encrypt）を有効化。
- **コスト**: ゼロ（ドメインは既存、APIキー・サーバー不要）。
- **配置（リポジトリ）**: `webapp/` にアプリ一式、`webapp/data/advice.json` にデータ。

```
webapp/
  index.html
  src/            … React コンポーネント・決定木・マッチング
  data/
    advice.json   … 自動生成された約163件のアドバイスDB
    taxonomy.json … 悩みカテゴリ階層（Q1→Q2→Q3）とタグ定義
  scripts/
    build-data.mjs … vault の md を読み advice.json を生成するバッチ
```

## 4. データモデル（advice.json の1件）

```jsonc
{
  "id": "169",
  "title": "20分の運動でADHDの脳が変わる",
  "author": "Pontifex",
  "year": 2013,
  "citation": "Pontifex MB, et al. ...",       // 論文citation（mdの1行目）
  "paperUrl": "https://pubmed.ncbi.nlm.nih.gov/...",
  "youtubeUrl": "https://youtu.be/...",          // 取得できた11本のみ。無ければ null
  "categories": ["発達特性", "生活習慣・環境"],   // Q1大カテゴリ（複数可）
  "tags": ["ADHD", "運動", "集中力", "小学生"],   // マッチング用キーワード
  "ageBands": ["小学生", "中高生"],
  "finding": "20分の有酸素運動の直後はADHD児の注意・読解課題の成績が上がった。", // 知見1文
  "advice": "勉強の前に20分だけ体を動かすと、集中しやすくなるかも。",            // 一言アドバイス
  "nextStep": "宿題の前に、なわとびや散歩を20分だけ試してみる。"               // 次の一歩
}
```

- `citation` / `paperUrl` / `title` は **mdから機械抽出可能**（既存の書式が安定）。
- `finding` / `advice` / `nextStep` / `categories` / `tags` / `ageBands` は
  **生成が必要**（タイトル＋論文情報、概要欄がある7本は本文も活用）。
- `youtubeUrl` は本文にある11本のみ自動取得。残りは §5.5 の方式で補う。

## 5. データ整備パイプライン（既存→JSON）

1. **抽出（自動）**: `scripts/build-data.mjs` が
   `東大理三夫婦の子育てエビデンス研究所/*.md`（136）＋ルート `NNN_*.md`（27）を走査し、
   id・title・citation・paperUrl・youtubeUrl を機械抽出して下書きJSONを作る。
2. **意味づけ（半自動・要レビュー）**: 各エントリに finding/advice/nextStep/
   categories/tags/ageBands を付与。**Claude Code が一括生成 → ユーザーが目視確認**。
   - 医療・発達系は誇張・断定を避け、「研究では〜」の表現で統一。
   - エビデンスの強さ（RCT/メタ分析/観察研究）を `evidenceLevel` として任意付与も可。
3. **確定**: レビュー済みデータを `advice.json` として確定し、アプリに同梱。

> ポイント: アプリは静的なので、データは「一度作って固定」。新作動画が出たら
> バッチを再実行＋追記レビューするだけで更新できる。

## 5.5 YouTubeリンクの方針（重要・要相談）

**前提となった制約**: 本実行環境では YouTube とブログ(risan.jpn.org)への
直アクセスがネットワークポリシーで遮断（403）され、かつ登録者数が少ない
チャンネルは一般のWeb検索にも出てこない。**よって、この環境から130本超の
動画URLを自動で正確に収集することはできない。** 取れているのは本文に
URLが書かれた11本のみ。

そこで、URLを全件持たなくても確実に「その動画」へ誘導できる方式を採用する:

- **方式A（推奨・即実装可）チャンネル内検索リンク**:
  各カードのYouTubeボタンを
  `https://www.youtube.com/@<チャンネルハンドル>/search?query=<動画タイトル>`
  にする。クリックすると、そのチャンネル内でタイトル検索した結果（=該当動画）に
  飛ぶ。必要な情報は **チャンネルの @ハンドル 1つだけ**。URL収集もメンテも不要。
  - 例: `.../@todai_ebe/search?query=20分の運動でADHDの脳が変わる`
- **方式B（最も正確）正確なURLを後から流し込む**: ユーザーが YouTube Studio の
  コンテンツ画面から「動画タイトル↔URL」をCSV等で書き出してくれれば、
  `youtubeUrl` を全件埋められる。方式Aと併用可（URLがあればそれを優先）。
- **方式C（折衷）**: まず方式Aで全件リンクを成立させて公開し、入手でき次第
  方式Bで正確URLに差し替える。← **本計画の既定とする**。

→ お願い: **チャンネルの @ハンドル（または /channel/ URL）を教えてください。**
  それだけで全カードのYouTube誘導が機能します。

  → **確定: チャンネルは `@evilab`**（ゆっくり子育てエビデンス研究所【東大理三夫婦】、
  動画176本）。YouTubeボタンは
  `https://www.youtube.com/@evilab/search?query=<動画タイトル>` を既定とする。
  正確URLが入手でき次第、該当エントリの `youtubeUrl` を差し替える（方式C）。

## 6. 悩みカテゴリ設計（コーパス実態にもとづく草案）

既存動画の主題から、大カテゴリは次の6つを提案（後で調整）:

1. **学習・勉強**（宿題Cooper / 認知負荷Sweller / メタ認知 / 例題効果 ほか）
2. **行動・しつけ**（体罰Gershoff / 報酬・ごほうびシリーズ / 問題行動 ほか）
3. **心・自己肯定感**（ほめ方Robichaud / 条件付きの愛Assor / レジリエンスWerner ほか）
4. **発達特性**（ADHD / 自閉症スペクトラム / 運動と脳Pontifex / CBT ほか）
5. **生活習慣・環境**（片付けFisher・Saxbe / 生活リズムEvans / 睡眠 / スマホ ほか）
6. **家族・きょうだい**（夫婦関係Gottman / 出生順位Sulloway / きょうだい ほか）

各大カテゴリにQ2（3〜5択）、必要に応じてQ3（年齢帯や状況）をぶら下げる。
※ タグ設計は build-data 段階でコーパスから語彙を吸い上げて確定する。

## 11. リスク・注意

- **医療/発達テーマの免責**: 「診断・治療の代替ではない」「気になる場合は専門家へ」
  を全結果に小さく常設。エビデンスの誤用・過度な一般化を避ける文面統一。
- **観察研究の因果断定回避**: 相関を因果のように言わない（advice生成時のルール）。
- **データ鮮度**: 静的データのため、公開後の更新はバッチ再実行が前提。
- **著作権/引用**: 論文要約は自作の知見文に留め、原文転載はしない。
